# ADR-001: QEMU Assumption Audit

## Status

Accepted â€” Phase 0 baseline.

## Context

BogKernel v16â€“v41 is explicitly QEMU-only. Every evaluator boots via
`qemu-system-i386 -kernel`. Device access uses hardcoded PC port addresses in
`kernel/bogk-kernel/src/main.rs`. Receipt strings encode `PLATFORM=qemu` and
`QEMU_ONLY=true`.

Bare-metal transition requires a complete inventory before any kernel changes.

## Decision

Maintain a machine-readable audit (`artifacts/qemu_assumption_audit.json`)
generated by `scripts/audit_qemu_assumptions.py`, synchronized with this ADR.
No kernel `#cfg` platform switches until Phase 1; default build stays QEMU-only.

## What Is Already Portable

- `bogk-core`: BOGVM executor, SHA-256, process/scheduler models, v40/v41
  genesis/journal, pseudo-FS table
- On-disk BogFS layouts (v37â€“v38): transport-independent; only ATA delivery is QEMU-tied
- Multiboot info struct parsing: standard spec
- Paging policy (CR3, user RO code): x86-specific but not QEMU-specific
- Syscall ABI v2 validation, IPC queues, `.bogapp` loader verification logic

## Consequences

- Phase 0 audit script must pass before Phase 1 kernel changes.
- Any new QEMU assumption must update this ADR and re-run the audit script.
- HAL extraction order follows ADR-003.
